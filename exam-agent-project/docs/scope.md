## Core Topic Clusters

### 1. Definition and Perspectives on Work

Students should understand that work is not only paid labor. In this course,
work is treated as a multi-level object that can be analyzed and redesigned.

Key concepts:

- Levels of work description: motion/action, task, process, work system
- Economic perspective: labor, productivity, paid and unpaid work
- Sociological perspective: division of labor, roles, authority, coordination
- Psychological perspective: performance, autonomy, feedback, task meaning
- Human factors and ergonomics perspective: human capability, fatigue, variability
- Operations and quality perspective: process, standardization, variation, improvement
- Information systems perspective: work system as the unit of analysis
- User/worker experience perspective: interface, interaction, usability, experience

Likely exam formats:

- Short definition question on work, task, process, and work system
- Compare-and-contrast question across disciplinary perspectives
- Application question asking students to analyze an example as work

### 2. Work Systems

Students should be able to describe a work system as an open socio-technical
system and analyze its components.

Key concepts:

- Work system as an open system
- Emergence: system-level performance arises from interactions among components
- Alter's Work System Framework
- Leavitt's Diamond
- ISO 6385 ergonomics framework
- Common components: purpose, outputs, customers/stakeholders, processes,
  participants, technology, information/rules, environment/infrastructure
- Work system distinction from isolated technology or individual task analysis

Likely exam formats:

- Identify components of a given work system
- Explain why performance is an emergent property
- Use a framework to analyze a familiar system such as a hospital, platform,
  factory, game, or AI service

### 3. Scientific Management as Work System Redesign

Students should understand scientific management as an early form of systematic
work system engineering.

Key concepts:

- Historical context: scientific reasoning applied to organized human work
- Taylor's diagnosis of rule-of-thumb work and systemic disorder
- Soldiering: natural and systematic soldiering
- Four principles of scientific management:
  - Develop a science for each element of work
  - Scientifically select and train workers
  - Promote cooperation through aligned incentives
  - Divide planning and execution responsibilities
- Taylor cases:
  - Cutting tool experiments
  - Pig iron handling at Bethlehem Steel
  - Shovel studies
  - Ball bearing inspection
- Gilbreth contributions:
  - Motion study
  - Therbligs
  - Fatigue reduction
  - Lillian Gilbreth and household/kitchen work systems
- Critical view: productivity gains, worker control, incentives, fatigue, and
  human-centered limitations

Likely exam formats:

- Explain the four principles
- Analyze one Taylor case as a work system redesign
- Discuss both strengths and limitations of scientific management

### 4. Anatomy and Taxonomy of Work Systems

Students should classify work systems and connect classification to suitable
improvement methods.

Key concepts:

- Anatomy of work systems:
  - Purpose / value proposition
  - Outputs
  - Customers / stakeholders
  - Processes and task structure
  - Participants and roles
  - Technology and tools
  - Information and rules
  - Environment and infrastructure
- Structure-based taxonomy:
  - Volume-variety: unit/small-batch, mass production, continuous process
  - Automation and agency: manual, mechanized, automated, agentic
  - Temporality: continuous/repetitive, project-based
- Complexity-based taxonomy:
  - Interaction complexity
  - Work domain complexity
- Operation-based taxonomy:
  - Transformation, transportation, storage, inspection, coordination,
    decision, service, and knowledge-oriented work

Likely exam formats:

- Classify a given work system using multiple taxonomy axes
- Explain why taxonomy matters for choosing improvement tools
- Compare manual, mechanized, automated, and agentic systems

### 5. Engineering Problem-Solving Process

Students should know the DASSI process and why structured problem solving is
important for engineering improvement.

Key concepts:

- DASSI:
  - Define the problem
  - Analyze the problem, including data collection
  - Search for possible alternatives
  - Select the best alternative through comparison
  - Implement the solution and follow up
- Problem definition: "a problem well defined is half solved"
- Symptoms versus root causes
- Existing method as one alternative
- Alternative comparison criteria: effectiveness, cost, feasibility, safety,
  time, and organizational compatibility
- Relation to PDSA, PDCA, and DMAIC
- Resistance to change and implementation barriers

Likely exam formats:

- Ask students to apply DASSI to a work system problem
- Explain why good ideas can fail during implementation
- Compare DASSI with another improvement cycle

### 6. Problem Structuring Methods

Students should understand methods for making sense of messy or ambiguous
problems before jumping to solutions.

Key concepts:

- KJ Method / affinity diagramming:
  - Gather language data such as facts, ideas, opinions, and issues
  - Group items by natural relationships
  - Discover unseen connections
  - Build shared understanding
  - Identify root causes and possible solution directions
- Concept Fan:
  - Expand problem framing by moving from a stated problem to broader purposes
  - Generate alternative ways to achieve the same purpose
  - Avoid fixation on one solution path

Likely exam formats:

- Explain when affinity diagramming is useful
- Use KJ logic to organize a messy problem statement
- Apply Concept Fan to reframe a narrow problem

### 7. Brainstorming and Ideation

Students should understand brainstorming as a structured method for divergent
thinking, not a casual discussion.

Key concepts:

- Classical brainstorming structure:
  - Ice-breaking
  - Idea purge phase
  - Idea trigger phase
  - Idea evaluation and solution recommendation phase
- Two fundamental principles:
  - Delayed judgment
  - Focus on quantity
- Basic rules:
  - No criticism
  - Short and snappy ideas
  - Avoid fixation
  - Accept unusual ideas
  - Combine, improve, and expand ideas
- Mental barriers:
  - False assumptions
  - Rigid solution strategies
  - Overcomplication
  - Information overload or scarcity
  - Evaluation apprehension
  - Fear and cultural/environmental barriers
- Cognitive interpretation:
  - Working memory limits
  - Long-term memory retrieval
  - Mode switching between creative and judgmental thinking
  - Writing and visualization as cognitive support

Likely exam formats:

- Explain delayed judgment and quantity focus
- Diagnose why a brainstorming session failed
- Design a better brainstorming session for a given engineering problem

### 8. Five Innovation Frameworks

Students should be able to identify and apply five systematic innovation
methods.

Key concepts:

- Addition framework: add a feature to improve utility, aesthetics, ergonomics,
  safety, or usability
- Subtraction framework: remove a component, step, or feature to simplify or
  improve a system
- Alternate means framework: achieve the same function through a different
  mechanism or technology
- Combination framework: combine existing technologies, services, or functions
  into a new solution
- Transposition framework: move a method, technology, or capability from one
  domain into another
- Use of generative AI and AI in finance as examples of these frameworks
- Evaluating innovation by usefulness, feasibility, integration, and commercial
  or operational value

Likely exam formats:

- Classify examples by innovation framework
- Generate improvement ideas using at least two frameworks
- Compare why one framework may be more suitable than another for a case

### 9. Micro-Level Motion Study and Therbligs

Students should understand motion study as a method for analyzing and improving
manual work.

Key concepts:

- Motion study: finding the preferred method of doing work
- Micro-level analysis versus process-level analysis
- Therbligs as basic motion elements developed by Gilbreth
- Important Therbligs:
  - RE / TE: reach or transport empty
  - G: grasp
  - M: move
  - RL: release
  - S: search
  - SL: select
  - P: position
  - PP: pre-position
  - A / DA: assemble and disassemble
  - AD / UD: avoidable and unavoidable delay
  - U: use
  - I: inspect
  - PL: plan
  - R: rest
  - H: hold
- Improvement ideas:
  - Reduce unnecessary distance
  - Improve tool layout
  - Use color coding, labels, guides, and pre-positioning
  - Reduce search, select, position, and avoidable delay motions
  - Distinguish worker-level issues from system-level workflow issues
- Principles of motion economy

Likely exam formats:

- Identify Therbligs in a manual task
- Propose improvements using motion economy principles
- Explain why Therblig analysis can reduce fatigue and improve efficiency
